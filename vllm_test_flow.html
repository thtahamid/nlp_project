8.2 — Test the vLLM Server

curl Requests · OpenAI API · 3 Test Endpoints

1CLI Client Terminal
Second SSH session (same instance)
Or remote machine with port 8000 open
Base URL: http://localhost:8000
2GET Health Check
GET /health
Verifies server is alive
Returns HTTP 200 if ready
Run first — if this fails, nothing works
2GET List Models
GET /v1/models
Lists registered model aliases
Should include arabic-ocr-v1
Confirms LoRA was loaded correctly
2POST Chat Completions
POST /v1/chat/completions
Sends base64 image + system prompt
"model": "arabic-ocr-v1" selects LoRA
max_tokens: 512
3SRV vLLM API Server
Receives HTTP request on port 8000
Routes to matching handler
Selects LoRA adapter from "model" field
Queues request for GPU execution
3REQ Request Structure
Content-Type: application/json
Body: system + user messages
User content: image_url type
URL: data:image/png;base64,...
4GPU Inference Execution
Vision encoder processes image tokens
LoRA weights applied to attention layers
Generates up to 512 tokens
PagedAttention manages KV-cache
5RSP /health Response
HTTP 200 OK
Body: {} (empty JSON)
Expected: immediate, no delay
Fail: server not started or crashed
5RSP /v1/models Response
{"data":[{"id":"arabic-ocr-v1",...}]}
Check: arabic-ocr-v1 is present
Fail: LoRA not loaded → recheck flags
Also shows base model listing
5RSP Inference Response
choices[0].message.content
Expected: {"transcription":"...","confidence":"high"}
Check: valid JSON with Arabic text
usage.completion_tokens visible
