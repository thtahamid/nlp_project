From scanned pages to model-ready JSONL

Arabic handwriting images are paired with transcriptions generated by gpt-5.4-mini (our reference labels — not gold-human labels). Each sample is then reshaped into Qwen's chat format, with the image embedded as a base64 data URL so the entire dataset is self-contained in two JSONL files.

LoRA on a 4-bit base — 44 min on one A10G

The base Qwen/Qwen2.5-VL-7B-Instruct is loaded in 4-bit NF4 with bf16 compute. A LoRA adapter (r=32, α=64, dropout=0.05) attaches to all attention + MLP projections — ~0.5% of parameters get trained, the rest are frozen. Three epochs, cosine schedule, effective_batch=2.

// findings · what the numbers say

Three things the run actually proved

Beyond the headline CER, the evaluation surfaced specific, well-characterized failure modes and at least one positive signal worth building on next iteration.

✓ pipeline works end-to-end Vision input reaches the decoder (base64 decode + Qwen chat template), JSON output is parseable 100% of the time, KV cache gives 11× latency vs. Run 01, and epoch-2 rollback validated the overfit hypothesis from the loss curve.
△ length bias dominates the mean Training references cluster long (median 61 chars) so the model learned an implicit "transcriptions are ~60 chars" prior. On short references it keeps generating, inflating CER via insertions rather than substitutions.
✗ LM prior leaks on weak vision When the vision signal is ambiguous, the decoder falls back to a canned Qwen-pretraining phrase (a 17-word Arabic sentence about negotiations) — the phrase has 0 matches in our training data, so it's a base-model prior, not a memorized artifact.
→ long-ref CER 0.829 is the real baseline On references > 60 chars, median and mean CER both sit at 0.829 with p90 at 0.903 — tight distribution, no catastrophic samples. That's the honest quality number for the next iteration to beat.

What Run 03 should change

The run-02 failure modes point to concrete, low-cost interventions. Ordered by expected return on effort:

1 · balance reference-length distribution Audit data/train/train.jsonl for short-reference coverage. If < 5%, augment with single-word samples so the model learns to terminate early instead of emitting its LM fallback.
2 · stop earlier · report honestly Keep stop_strings=['"}'], report stratified CER as the primary metric (short / med / long) instead of a single mean, and optionally clamp to min(CER, 1.0) for headlines.
3 · drop epoch 3 · add regularization Train for 2 epochs (not 3) given the overfit signal; add weight_decay=0.01 or raise LoRA dropout to 0.1 if a third epoch is retained.
4 · push long-ref CER below 0.5 With balanced data and proper stopping, the goal for Run 03 is bringing the long-ref bucket from 0.829 → < 0.5; that's the regime where exact-match starts becoming non-zero.
